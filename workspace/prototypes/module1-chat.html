Auth 模块技术方案
版本 1.2 | 2026-04-19 | 陈朗辉
1. 认证方式: JWT (RS256)
- Access Token 有效期: 24h
- Refresh Token 有效期: 7d, 支持 rotation
2. API 设计
POST /api/auth/login → { access_token, refresh_token }
🤖
代码助手 14:30
这个 API 设计需要考虑并发: 如果两个客户端同时用相同的 refresh token 请求, 需要有 token rotation 的原子性保证。建议使用 Redis SETNX 做 token 消费锁。
陈
陈朗辉 14:35
好建议, 加上。另外 token family 检测也需要: 一旦 refresh token 被重用, 立即撤销整个 token family。
POST /api/auth/refresh → { access_token, refresh_token }
POST /api/auth/logout → { success: true }
