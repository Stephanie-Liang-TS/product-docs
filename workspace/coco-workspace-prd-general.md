## 1. 竞品分析

### 1.1 竞品概览

我们调研了 10 款主流 AI 团队协作产品，从团队协作模型、Agent 持久化、共享记忆、隐私模型、定价策略五个维度进行了深度对比。

---

### 1.2 逐一分析

#### 1.2.1 ChatGPT Team / Enterprise (OpenAI)

| 维度 | 详情 |
|------|------|
| **团队协作** | Team 提供共享工作区（Shared Workspace），成员可发布和共享自定义 GPT；Enterprise 增加更细粒度的工作区管理 |
| **Agent 持久化** | 无持久化 Agent，对话为即时会话模式，用户关闭即停止 |
| **共享记忆** | 无跨用户共享记忆；每个用户有独立的 Memory 功能，但不互通 |
| **隐私模型** | Team 不使用用户数据训练模型；Enterprise 额外提供 SOC 2 合规、数据加密、自定义保留策略 |
| **定价** | Team $25/用户/月（年付）；Enterprise 自定义定价 |
| **独特之处** | 最大的用户基数、GPT Store 生态、支持 GPT-4o/o1 等全线模型 |

**对 COCO 的启示**: ChatGPT 的团队产品本质仍是"多人各用各的"，没有真正的协作 Agent。这恰好是 COCO 可以差异化的核心空间。

---

#### 1.2.2 Claude Team / Enterprise (Anthropic)

| 维度 | 详情 |
|------|------|
| **团队协作** | Team（~$30/用户/月）支持共享 Projects，团队成员可协作使用同一 Project 的知识库和系统提示词 |
| **Agent 持久化** | 无持久化 Agent；所有交互为即时会话 |
| **共享记忆** | Projects 功能提供有限的共享上下文（文件+提示词），但无跨对话的长期共享记忆 |
| **隐私模型** | 不使用团队数据训练模型；Enterprise 提供 SAML SSO、SCIM、审计日志、HIPAA 合规 |
| **定价** | Team ~$30/用户/月；Enterprise 自助版和销售版两种；团队上限约 75 席位后推荐 Enterprise |
| **独特之处** | Projects 的知识库共享是最接近"共享记忆"的竞品实现 |

**对 COCO 的启示**: Claude Projects 验证了"共享知识库"的市场需求，但它仍停留在静态文档层面，缺乏 Agent 的自主学习和记忆积累。COCO 的动态共享记忆是本质差异化。

| 维度 | 详情 |
|------|------|
| **团队协作** | Business 提供组织级管理和 Copilot 策略控制；Enterprise 提供 Copilot Spaces（共享上下文空间：代码+文档+笔记） |

### 1.4 竞品分析关键发现

1. **"共享记忆"是蓝海**: 除 Notion AI 天然依托知识库外，几乎没有竞品实现了真正的跨用户动态记忆共享。大多数产品的"团队功能"只是"共享账单 + 管理面板"。

2. **"Agent 持久化"正在成为趋势**: Taskade、Lindy、Notion Custom Agents 都在往 always-on 方向走。COCO 在这方面已有先发优势（人停 Agent 不停）。

3. **"AI 员工"定位极度稀缺**: 只有 Lindy 和 COCO 走"专属 AI 员工"路线。Lindy 主打个人场景，缺乏团队协作。**COCO Workspace = 团队共享 AI 员工**，这是全市场独一无二的定位。

4. **定价集中在 $25-$40/用户/月**: 这是"AI 工具增值"的价格带。COCO 定价更高（$99-$999），对应的是"AI 员工替代人力"的价值主张，需要维持这一定位差异。

5. **隐私分层是共识**: 几乎所有竞品都在 Team 层不训练数据，Enterprise 层提供 SSO/SCIM/审计日志/合规认证。

## 2. 四项关键决策

### 决策 A：个人订阅如何与 Workspace 共存

#### 推荐方案：订阅绑定到 Workspace，个人用户自动拥有一个"个人 Workspace"

**具体设计**：
- 每个付费用户默认有一个**个人 Workspace**（1 人 workspace），其订阅等级决定该 workspace 的资源上限
- 用户可以**创建或加入团队 Workspace**，团队 Workspace 有独立的订阅和计费
- 个人 Workspace 和团队 Workspace **共存不冲突**——用户在不同 Workspace 间切换
- 个人订阅可以**升级转化为团队 Workspace**（保留所有数据和记忆）

**理由**：
1. **参考 Notion 模式**: Notion 的个人免费版和团队版完美共存，用户同时拥有个人空间和团队空间，这一模式被验证非常自然
2. **降低迁移门槛**: 现有付费用户零成本过渡——他们的账号自动变成"个人 Workspace"，功能完全不变
3. **自然升级路径**: 个人用户邀请第一个队友时，就产生了升级团队 Workspace 的需求
4. **不破坏现有营收**: 现有订阅完全保留，团队 Workspace 是增量收入

**迁移方案**：
- 现有 Air/Pro/Ultra 用户 → 自动创建同级别个人 Workspace
- 不做强制迁移，个人 Workspace 在功能上与当前体验完全一致

---

### 决策 B：Workspace 内私聊的默认隐私模型

#### 推荐方案：私聊内容默认仅自己可见，Agent 知识双轨制

**具体设计**：
- **私聊内容**：用户与 Agent 的 1:1 对话，**默认仅发起者可见**，其他 Workspace 成员（包括 admin）不可见对话内容
- **知识双轨制**：
  - Agent 从私聊中学到的**事实性知识**（如"客户 A 的合同金额是 X"）→ 写入 Workspace 共享记忆，标记来源
  - 对话中的**个人观点、情绪表达、私密讨论**→ 仅存入个人记忆层，不共享
  - 用户可以手动将私聊中的信息**标记为共享**或**标记为私密**，覆盖默认行为
- **Admin 权限**：Admin 可以看到私聊的**存在**（"张三与 Agent 有 47 次对话"），但不能看到**内容**；Enterprise 可按合规需求开放审计权限

**理由**：
1. **信任是基础**: 如果用户觉得"我跟 AI 说的话老板都能看到"，他们就不会真实沟通，Agent 的价值会大打折扣
2. **参考 Slack 模式**: Slack DM 对 admin 也不默认可见（虽然 Enterprise 可以导出），这是被广泛接受的隐私标准
3. **知识共享 ≠ 对话共享**: Agent 的价值在于**积累和分享知识**，而非暴露对话过程。"知识双轨制"让 Agent 既能为团队积累知识，又保护个人隐私
4. **合规灵活性**: Enterprise 层面保留审计能力，满足合规要求

---

### 决策 C：单 Agent vs 多 Agent per Workspace

#### 推荐方案：MVP 单 Agent，Phase 2 引入专业化 Agent 角色

**具体设计**：
- **Phase 1（MVP）**：每个 Workspace **1 个 Agent**，所有成员与同一个 Agent 交互
  - Agent 拥有统一的人格、记忆和能力
  - 通过"频道"（Channel）区分不同工作上下文（类似 Slack 频道），但 Agent 是同一个
  - Agent 可以在频道间共享信息（尊重隐私规则）
- **Phase 2**：引入 **Agent 角色**（Roles），单 Agent 可切换角色
  - 例如"项目经理模式"、"客服模式"、"数据分析模式"
  - 角色影响行为偏好和专业知识加载，但共享同一个核心记忆
- **Phase 3（远期）**：支持 **多 Agent**
  - Workspace 可创建多个独立 Agent，各有独立人格和专业领域
  - Agent 间可以相互通信和协作

**理由**：
1. **复杂度控制**: 多 Agent 的技术复杂度极高（Agent 间通信、记忆隔离/共享、冲突解决），MVP 不应承担这个风险
2. **用户认知负担**: "我有一个 AI 员工"比"我管理一个 AI 团队"更容易理解和接受。参考 Lindy 也是从单 Agent 开始的
3. **竞品验证**: Taskade 和 Relevance AI 虽然支持多 Agent，但用户实际使用场景显示，大部分团队只活跃使用 1-3 个 Agent
4. **角色是轻量替代**: 很多"多 Agent 需求"本质是"同一个 Agent 在不同场景表现不同"，角色系统可以低成本满足 80% 的需求
5. **商业节奏**: 多 Agent 可以作为高阶套餐的差异化功能，为后续涨价提供空间

---

### 决策 D：计费模型

#### 推荐方案：Workspace 订阅制 + 按席位阶梯定价

**具体设计**：

| 层级 | 目标用户 | 定价 | 核心差异 |
|------|---------|------|---------|
| **Personal** | 个人/自由职业者 | 保持现有 Air $99 / Pro $449 / Ultra $999 | 个人 Workspace，1 Agent，1 用户 |
| **Team** | 小团队（2-10人） | $79/用户/月（最低 3 席位起） | 团队 Workspace，共享记忆，协作对话，基础管理 |
| **Business** | 中型团队（10-50人） | $59/用户/月（最低 10 席位起） | 全部 Team 功能 + SSO + 审计日志 + Agent 角色 + 优先支持 |
| **Enterprise** | 大型组织（50+） | 自定义 | 全部功能 + SCIM + 合规认证 + 专属客户成功 + SLA |

**定价逻辑**：
1. **席位越多单价越低**: 激励团队扩张（$79 → $59 的阶梯本身就是激励）
2. **远高于竞品的 AI 工具定价**: 因为 COCO 不是"AI 辅助工具"，而是"AI 员工"。$79/用户/月 ≈ $948/用户/年，对比招聘一个初级员工的成本（$40K-$60K/年），ROI 极其明显
3. **个人计划不动**: 现有营收基本盘不受影响
4. **最低席位门槛**: 确保团队计划有足够的 ARPU，避免"两人团队占着团队功能"的问题

**对现有用户的影响**：
- 个人用户：零影响，计划和价格完全保留
- 想协作的用户：新建团队 Workspace 或升级，原有个人 Workspace 可保留
- 账单简化：Workspace 级统一账单，一个 Workspace 一张发票

**理由**：
1. **对标市场但保持溢价**: 竞品团队计划 $25-$40/用户/月，COCO $79 的溢价对应"AI 员工"价值主张。参考 Lindy Pro $99.99/月（个人），COCO 团队价 $79 比 Lindy 个人价还低，有竞争力
2. **席位制是 SaaS 标准**: 客户理解、可预测、易管理。避免 credits/tokens 等复杂计量模式（Notion AI 的 credits 模式被用户普遍吐槽）
3. **最低席位保底**: 3 人起步 = $237/月最低 ARPU，确保团队客户的服务成本可覆盖

## 3. 产品需求文档

### 3.1 产品概述

**COCO Workspace** 是 COCO AI 员工产品的团队协作层。它解决的核心问题是**"人停 Agent 停"**——当前模式下，Agent 的生命周期绑定在单个用户上，用户离线则 Agent 闲置，无法服务团队其他成员，也无法持续执行后台任务。

Workspace 将 Agent 从"个人工具"升级为"团队成员"：
- **Agent 生命周期独立于任何单一用户**——团队中任何人都可以与 Agent 交互，Agent 7x24 持续工作
- **共享记忆**——Agent 积累的知识为整个团队服务，不因人员变动而流失
- **协作原生**——支持多人与 Agent 的协作对话、任务委派和信息流转

**产品愿景**: 让每个团队拥有一个永不下线、持续进化、了解所有上下文的 AI 员工。

### 3.3 用户故事

#### 高优先级用户故事

**US-01: 创建团队 Workspace**
> 作为一个 COCO 用户，我希望能创建一个团队 Workspace 并邀请队友加入，这样我们可以共享同一个 AI 员工。

验收标准：
- 用户可以创建 Workspace 并设置名称
- 可通过邮件/链接邀请成员
- 被邀请人接受后立即可与 Agent 交互
- Workspace 创建者自动成为 Admin

**US-02: 与 Agent 协作对话**
> 作为 Workspace 成员，我希望在公共频道中与 Agent 对话时，其他成员可以看到对话内容并参与，这样我们可以协作解决问题。

验收标准：
- 公共频道中的对话对所有成员可见
- 任何成员可以在公共对话中接续提问
- Agent 回复时综合考虑所有参与者的输入
- 对话中标注每条消息的发送者

**US-03: 私密对话**
> 作为 Workspace 成员，我希望可以与 Agent 进行私密 1:1 对话，其他成员看不到对话内容。

验收标准：
- 用户可以发起与 Agent 的私密对话
- 对话内容仅该用户可见
- Agent 可以将私聊中的通用知识（非个人观点）写入共享记忆
- 用户可以手动控制哪些信息共享/不共享

**US-04: Agent 持续工作**
> 作为团队负责人，我希望 Agent 在所有人离线后仍然可以继续执行任务，这样早上上班时可以看到任务结果。

验收标准：
- Agent 收到任务后持续执行，不因用户下线停止
- 任务完成后通过配置的通知渠道（Telegram/Lark/邮件）主动通知
- 支持查看 Agent 的任务执行历史和状态

**US-05: 共享记忆**
> 作为新加入的团队成员，我希望 Agent 已经了解团队的项目背景、决策历史和工作偏好，这样我不需要从头介绍一切。

验收标准：
- 新成员加入后，Agent 即具备 Workspace 中的共享知识
- 共享记忆包括项目信息、决策记录、团队偏好等
- Agent 能区分"团队共识"和"个人偏好"

**US-06: 权限管理**
> 作为 Workspace Admin，我希望能管理成员权限，控制谁能修改 Agent 配置、谁只能对话。

验收标准：
- 支持至少 3 个角色：Admin / Member / Viewer
- Admin 可管理成员、配置 Agent、查看用量统计
- Member 可与 Agent 对话、创建频道
- Viewer 只能查看公共频道内容

#### 中优先级用户故事

**US-07: 统一账单管理**
> 作为 Workspace Admin，我希望整个团队的费用统一出账，而不是每个人单独付费。

**US-08: Agent 行为自定义**
> 作为 Workspace Admin，我希望可以设定 Agent 的工作规则和偏好（如回复风格、工作时间、专业领域），让 Agent 符合团队需要。

**US-09: 消息归因**
> 作为 Workspace 成员，我希望在查看 Agent 的回复时，知道它引用了谁的输入或哪条记忆，这样我可以追溯信息来源。

**US-10: 从个人计划迁移**
> 作为现有 COCO 个人用户，我希望把现有 Agent 的记忆和配置迁移到新的团队 Workspace，不丢失任何数据。

#### 低优先级用户故事

**US-11: Agent 角色切换**（Phase 2）
> 作为 Workspace 成员，我希望 Agent 可以在不同角色（项目经理/数据分析师/客服）之间切换，在不同频道表现不同的专业能力。

**US-12: 跨 Workspace 切换**
> 作为属于多个 Workspace 的用户，我希望方便地在不同 Workspace 之间切换，每个 Workspace 有独立的 Agent 和记忆。

### 3.5 四个决策结论及理由

#### 决策 A：个人订阅与 Workspace 共存

**结论**: 订阅绑定到 Workspace，个人用户自动拥有"个人 Workspace"。团队 Workspace 独立订阅。两种 Workspace 共存。

**理由**:
- 参考 Notion、Slack 成功模式——个人空间 + 团队空间自然共存
- 对现有付费用户零干扰，自动平滑过渡
- 创造清晰的个人→团队升级路径（邀请队友即触发）
- 不破坏现有 $99-$999/月的个人收入基本盘

#### 决策 B：私聊默认隐私模型

**结论**: 私聊内容默认仅自己可见。Agent 采用"知识双轨制"——事实性知识进共享记忆，个人观点留私有层。Enterprise 保留合规审计能力。

**理由**:
- 信任是 AI 员工价值的基础——用户必须敢说真话
- 知识共享 ≠ 对话暴露，这是关键的产品洞察
- 参考 Slack DM 隐私标准，被市场广泛接受
- 分层合规设计满足从初创到企业的全谱需求

#### 决策 C：单 Agent vs 多 Agent

**结论**: MVP 单 Agent per Workspace。Phase 2 引入 Agent 角色系统。Phase 3 考虑多 Agent（Enterprise 功能）。

**理由**:
- 单 Agent 用户认知最简单——"我们团队有一个 AI 员工"
- 角色系统可低成本满足 80% 的"多 Agent"需求
- 控制 MVP 技术复杂度，加速上线
- 多 Agent 保留为高阶 Enterprise 差异化功能

#### 决策 D：计费模型

**结论**: Workspace 订阅制 + 按席位阶梯定价。Team $79/用户/月（3 人起）、Business $59/用户/月（10 人起）、Enterprise 自定义。个人计划不变。

**理由**:
- "AI 员工"价值主张支撑高于竞品的定价（竞品 $25-$40 是"AI 工具价"）
- 席位制是 SaaS 标准，客户理解成本低
- 阶梯定价激励团队扩张，提升 LTV
- 最低席位保底确保 ARPU 覆盖服务成本

### 3.7 技术边界和约束

#### 已知约束

1. **Agent 运行时成本**: 每个 Workspace 的 Agent 需要持续运行的计算资源。初期可能需要限制 Workspace 数量或通过排队机制管理资源
2. **模型上下文窗口**: 共享记忆越多，单次对话可加载的上下文有限。需要设计记忆检索和摘要机制，而非全量加载
3. **多用户并发**: 多人同时与同一个 Agent 对话时，需要消息队列和优先级机制，避免 Agent 响应混乱
4. **跨平台集成**: Agent 需要同时响应 Telegram、Lark、Web Console 等多渠道消息，Workspace 内的频道模型需要与现有 C4 通信桥兼容
5. **记忆一致性**: 共享记忆的读写并发需要保证一致性，避免"幻觉记忆"（Agent 引用了被其他人删除的记忆）

#### 技术依赖

| 依赖 | 说明 |
|------|------|
| 现有 Agent 基础设施 | Workspace 建立在当前 zylos 运行时之上，需要扩展为多租户架构 |
| 用户认证系统 | 需要新建或集成身份认证服务，支持邀请/加入流程 |
| 计费系统 | 需要扩展 Stripe 集成，支持 Workspace 级订阅和多席位管理 |
| 消息路由层 | 需要改造现有 C4 通信桥，支持 Workspace/频道级消息路由 |
| 记忆存储 | 需要设计新的记忆存储架构，支持个人记忆层和共享记忆层 |

#### 不在范围内

- 自定义模型训练/微调（用户不能用自己的数据训练 Agent）
- 本地部署/私有化部署（MVP 阶段仅支持 SaaS）
- 跨 Workspace Agent 协作（不同 Workspace 的 Agent 不互通）
- 实时语音/视频与 Agent 交互

## 附录

### A. 术语表

| 术语 | 定义 |
|------|------|
| **Workspace** | COCO 中的团队协作空间，包含 Agent、成员、频道和共享记忆 |
| **个人 Workspace** | 单人使用的 Workspace，对应现有个人订阅 |
| **团队 Workspace** | 多人使用的 Workspace，对应新的团队/商业/企业订阅 |
| **频道（Channel）** | Workspace 内的对话空间，类似 Slack 频道 |
| **共享记忆** | Workspace 级别的知识存储，所有成员通过 Agent 共享 |
| **知识双轨制** | Agent 将私聊中的事实性知识写入共享记忆、个人观点留在私有层的机制 |
| **Agent 角色** | Agent 的专业化配置，定义特定场景下的行为和知识 |
| **消息归因** | 标注每条消息发送者身份的机制 |

### B. 风险与缓解

| 风险 | 影响 | 概率 | 缓解策略 |
|------|------|------|---------|
| Agent 运行成本高于预期 | 毛利被压缩 | 中 | 实施智能调度——空闲时降低运算频率；设置每 Workspace 资源上限 |
| 团队用户增长慢于预期 | 研发投入回报周期长 | 中 | Phase 1 控制研发投入；优先转化现有活跃个人用户 |
| 共享记忆导致信息泄露 | 用户信任危机 | 低 | 知识双轨制 + 明确的隐私 UI + 用户手动控制 |
| 多人并发导致 Agent 混乱 | 用户体验差 | 中 | 消息队列 + 对话上下文隔离 + 压力测试 |
| 竞品快速跟进 | 差异化窗口缩短 | 高 | 加速 MVP 上线；深耕"AI 员工"人格化差异 |
